# Frontend mood pages（沉浸基调页）

单页 HTML 作品：情绪与画意优先，**华丽、强动态、反极简**；默认多层叠效与错频动画，而非大留白或系统字体模板。

## 激发本 Skill 的关键词（显式列表）

以下表述（及相近说法）应优先按本 Skill 执行；用户 **@frontend-mood-pages** 或写明「用沉浸基调页规范」时**必须**遵循本文件。

### 中文

- **页面类型**：基调页、氛围页、沉浸页、全屏页、全屏沉浸 HTML、游戏/剧情向单页
- **布局**：拼图格、联画、多格拼图、三列拼图、三联画、横联画、宫格、分屏画意
- **动效与层次**：画意动效、氛围动效、叠层动效、照片上动效、分区动效
- **文字**：叠字、画上文字、哲学短句、金句叠在图上（常配合英文斜体）
- **风格指向**：唯美、黑深残、圆满、暗金文学风、油画感网页（与仓库范例同类）

### English

- `mood page`, `tone page`, `immersive full-screen`, `full-viewport HTML`
- `triptych`, `grid collage`, `panel layout`, `split-screen art layout`
- `layered CSS effects over photos`, `animation on imagery`
- `literary overlay`, `text over image`, `epigraph on panel`
- `ornate`, `animation-heavy`, `non-minimal` single-page (when tied to art/game tone)

**不触发本 Skill**：幻灯片、deck、每页 100vh 分页、演讲稿式结构 → 用 **frontend-slides**。

## Phase 0（强制）：需求澄清与门禁

**在用户对必问项作出回答，或明确确认你的汇总之前，不得编写或修改 HTML/CSS/JS，不得改图片资源**（含将 **未确认** 的文学层 `color` / `text-shadow` / `font-family`，以及 **未逐图确认** 的画意动效——格内 `fx-*` 定稿、`@keyframes` 绑定与时长——写入实现稿）。只能提问、归纳、等待。

### Cursor Plan 模式与本 Skill（强制，须强调）

- **无论当前对话是否处于 Cursor Plan 模式**，只要按本 Skill 产出基调页方案或实现，**必须执行与 Agent 模式同等完整度的询问链**：**主题 → 图片审查 → fit 态度 → 逐图色系 → 排版选项 → 字体与文学层色彩 → 逐图画意动效**。**禁止**因 Plan 模式而只做「纯架构不写追问」、或明显缩减 Phase 0 提问粒度。
- **须向用户明确强调**：**Plan 模式也必须询问；询问点与 Agent 模式一致；用户未确认前不得进入实现**（含不得将未确认的 `color` / `font-family` / 格内 `fx-*` 定稿写进计划或代码）。
- 若用户在**单条消息**中已一次性给出 Phase 0 全部要点并写明「确认」或等价表述，可先 **复述汇总** 并请其 **书面确认** 后再继续计划或编码；否则仍须按上列顺序 **补齐提问**。

### 图片审查（体积、像素与整张入画）— 有素材时必做

在用户给出图片路径或文件夹后，**未走完下列审核链（或用户明确接受「尺寸未知 + 裁切/全显态度」的书面确认）之前，不得输出「排版询问格式」中的 A/B/C 方案。**

**审核顺序（必须遵守）**

1. **列出**将用于页面的图片文件（扩展名如 jpg/png/webp 等）。
2. **文件体积**：用环境允许的方式读取每个文件大小（如终端 `Get-Item` / `dir` / `stat`）。单张 **明显偏大**（例如大于约 1.5–2 MB）时：在 Phase 0 **告知用户**，并建议 Web 侧可选方案（压缩、降分辨率、改 WebP），由用户决定是否仍用原图。
3. **像素尺寸与宽高比（工具可读时视为必做）**  
   - 记录每张 **宽 × 高**（px），写出 **宽高比**（如 16∶9、4∶5、3∶4）。  
   - 将图源比例与 **常见视口**（横屏约 16∶9、竖屏约 9∶16）及你**初步设想格区**的宽高比对比：相差大时，`object-fit: cover` 会 **大幅裁边**；须在后续建议中点明，并优先用「比例匹配」或 `contain`+衬底（见下节）。  
   - **工具不可读**时：在对话中写明「像素尺寸未知」，并 **追问**：是否 **必须整张可见**、可接受的裁切（上/下/侧缘/仅中心氛围）。**禁止**在尺寸未知时断言「一定放得下」。
4. **每张图的色系**：见下一节；**色系未逐张落实前，同样不得给出 A/B/C**。

将审查结论（体积摘要、各图尺寸与比例、与视口/格区关系的判断、色系摘要）**写进**排版建议的附言中（不代替用户最终决定）。

### 整张入画与 `object-fit` 策略（显式）

技术实现里常用 `cover` 填满格区，易裁切主体。**用户强调整张入画、或图源与格区比例严重不符时**，助手须在 Phase 0 与用户对齐策略，并在 **每条**排版方案中 **按文件名（或图 1、图 2…）** 标注拟用 **`cover` 还是 `contain`**；若 `cover`，简述预计 **主要裁哪一侧** 或是否「以中心为主」。

| 策略 | 含义 | 适用 |
| ---- | ---- | ---- |
| **比例匹配** | 该图所在格区（或整页单图）的 CSS 宽高比（`aspect-ratio` 或行/列 `fr` 组合）尽量贴近图源，仍用 `cover` 但裁切最小 | 横图横条、竖图竖列、单图全屏 |
| **contain + 画意衬底** | `object-fit: contain`，letterbox 区域用渐变、微粒、纹理、与邻格同色系铺底（**禁止**大块死白极简，须符合本 Skill「华丽、反极简」） | 必须见全图、或多张比例混乱 |
| **接受裁切** | 用户确认该张只需氛围，`cover` 可裁主体外区域 | 装饰性底图 |

### 每张图的色系（给出排版选项之前必须完成）

**在输出任何排版方案（A/B/C 或等价选项）之前**，须已完成 **本节 + 上文「图片审查」第 1～3 步**（或用户已确认尺寸未知与裁切/全显态度）。针对**每一张**将入页的图，弄清 **主色系 / 冷暖 / 明暗**（用于拼缝两侧是否冲突、哪几张适合相邻、全页底色与动效叠色倾向）。

1. **询问方式**：按文件名（或图 1、图 2…）逐项问用户；或你先 **Read 读图** 写出一行「主色倾向」摘要，再请用户 **确认或改正**。
2. **最少信息**：每张至少明确：偏暖/偏冷/中性、整体亮/中/暗、1～2 个主色关键词（如「橙金晚霞」「青蓝海面」「墨绿林」）。
3. **与排版的关系**：强对比色系（如暖霞 vs 冷海）可考虑 **对角或远距** 分格，避免窄缝硬贴；相近色系可 **相邻** 强化连续氛围；极暗图占大块时考虑与较亮图 **平衡格比**。
4. **门禁**：色系未逐张落实（或用户未确认你的读图归纳）前，**不得**进入「排版询问格式」一节去给出 A/B/C；只能继续追问或等待。

### 字体与文学层色彩（用户拍板）

**目标**：叠字与标题色不默认收敛为单一「暗金 + Georgia」；须结合 **主题** 与 **已确认的逐张图色系**，在 Phase 0 给出 **2～3 套并列候选**（每套：字体气质一句 + 主字色相倾向 + `text-shadow` 可读策略一句）。**候选不是定稿**。

**门禁**：未获用户对 **展示字体**（具体 Google Fonts 族名，或明确「仅用 Georgia / Times 系」）、**主字色**（`#` 或 `rgba`）、**`text-shadow` 层次**（亮边/暗描边/晕光的大致强度，及是否采用某套候选中的 shadow 色倾向）、**是否按格变色**（全页统一 vs 每格不同）的 **选定或书面确认** 前，**不得**在实现稿 HTML/CSS 中将任何字色、`text-shadow` 数值、`font-family` 当作最终定稿写死。可先仅在对话中列出候选。

**流程**：

1. 在 **主题 + 逐张图色系已确认** 之后（可与排版 A/B/C 同轮或紧随其后），根据整体冷暖/明暗抛出 **2～3 套** 方向，例如下表中的类型（**允许改写文案，禁止只给一套**）：

| 类型代号 | 气质与常见字体倾向 | 主字色与 shadow 倾向（仅描述，非定值） |
| -------- | ------------------ | ---------------------------------------- |
| 暖金文学 | 衬线、常配 italic；Playfair / Cormorant 类 | 琥珀金、蜜褐；shadow 深褐描边 + 淡金高光 |
| 冷银雾青 | 偏细或偏锐衬线；EB Garamond / Cormorant 类 | 灰蓝、冰青、银白；shadow 深蓝灰描边 + 浅青晕 |
| 紫金戏剧 | 偏戏剧展示衬线；Cinzel / Cormorant 等 | 淡香槟字 + 紫金/茄色描边；适合暗底 |
| 森绿墨线 | 略偏文艺衬线 | 苔绿、墨绿字；shadow 近黑绿描边 + 极弱黄绿高光 |
| 尘玫瑰 / 枯荷 | 柔和衬线 | 灰玫瑰、藕色；shadow 褐灰描边，避免纯灰字无层次 |
| 象牙高对比 | 亮底或高亮图上 | 近黑褐、深茶字；shadow 白或奶油高光为主 |

2. 用户 **选定一套或混搭**（如「套 B 的字体 + 我自己主色 #…」）后，**仍须**落实 **具体色值** 与 **是否每格不同**；若用户直接给出完整 CSS 颜色与字体列表，以用户为准。

3. **禁止**：因「惯例」或图省事，把「叠字配方」中的演示 `rgba(...)` 或示例 `font-family` 直接抄入页面当作定稿，**除非**用户已确认采用或等价数值。

### 每图画意动效（用户拍板）

**目标**：每一张入页图所在格内，须有 **明显、可感知** 的画意辅助动效（见下节「画意动效类型词典」与「每图独立画意层」）；动效 **类型与强度不得由助手默认定稿**，须 **逐图** 经用户 **同意或修改指定** 后再写入实现稿。

**流程**（建议在布局方案、fit、字体与文学层色彩已清晰后执行，也可与字体色同轮合并，但须 **每张图单独一行/一条** 说清楚）：

1. 按 **文件名（或图 1、图 2…）** 逐项：根据该图色系与明暗，从 **画意动效类型词典** 给出助手 **推荐 1 套**（可选 **2～3 个替代候选**），并用一句话解释 **为何辅助该图**。
2. 用户 **选定、改写或指定**（如「图 A 要扫光不要微粒」「图 B 动效再强一点」）。若用户一次性给出逐表（图名 → 类型 → 强度），以用户为准。
3. **汇总表（建议）**：实现前在对话中列出表格：**图名 | 选定动效类型（可写组合）| 强度（强 / 中 / 柔）| 是否与 `img` 浮动并存**，经用户 **确认** 后再编码。

**门禁**：**未完成全部入页图** 的画意动效方案 **用户确认** 前，不得在实现稿中为任意一格写定 **最终的** `@keyframes` 名称绑定、`animation-duration` / `delay` 定稿，或格内 `fx-*` 的 **最终** 结构与层数（对话中的文字描述允许）。

**例外**：仅当用户在 Phase 0 **书面声明**「某格不要动效」或「全页只要极简」时，可对该格省略；默认仍须 **逐图问清**，不得擅自省略。

### 必问（最低集）

用 **AskQuestion**（若可用）或一轮对话尽量合并提问（顺序建议：**主题 → 图片审查（列文件、体积、像素与宽高比）→ 整张入画 / 裁切态度（尤其尺寸未知时）→ 每张图色系 → 排版选项 → 字体与文学层色彩（用户拍板）→ 每图画意动效（用户拍板）**）：

1. **主题 / 基调**：情绪关键词（如唯美、黑深残、圆满）、参考系（是否对齐仓库内某页）、文案语言（如英文斜体叠字）、禁忌（如不要 blur 接缝）。
2. **图片审查与 fit 态度**：体积与像素比例（见上）；若用户要求全图可见，须对齐 **比例匹配** 或 **contain + 画意衬底**。
3. **每张图的色系**（在排版前完成）：见上节。
4. **布局与结构**（须含 **助手建议选项 + 用户自定义**）：见下节；**仅**在审核链完成后输出。
5. **字体与文学层色彩**：见上节「字体与文学层色彩（用户拍板）」；**须在编写实现稿前**得到用户对字体族、主字色、shadow 层次、是否按格变色的确认。
6. **每图画意动效**：见上节「每图画意动效（用户拍板）」；**须在编写实现稿前**得到用户对 **每一张图** 的动效类型（或组合）、强度及汇总表的确认。

### 排版询问格式（建议选项 + 自定义）

**输出时机**：仅当 **图片审查**（含体积、像素宽高比或用户已确认尺寸未知与裁切/全显态度）与 **每张图色系** 均已落实后，才给出下列方案。

询问布局时 **禁止只问开放题**。应：

1. 根据 **主题 + 图片审查结果 + 每张图色系**，给出 **2～3 条具体布局方案**（每条说清楚：结构类型、几行几列/主从分区、哪块跨格、无缝或有缝、移动端是否纵排或是否保留斜切），并 **点明**色系关系（如「暖冷对角」「相邻渐变」）。
2. **每条方案必须附带「按图 fit」附言**：对将入页的每张图写明拟用 **`cover` 或 `contain`**；若 `cover`，根据格区比例与图源比例 **推断**并写明预计 **主要裁切方向**（如「竖图置于扁横格 → 上下易裁」）或「中心保留」。
3. **必须包含最后一项**：例如 **「自定义：我将在下一条消息里描述布局」** 或 **「其他（请说明格数、比例、移动端、各图 cover/contain）」**。
4. 若用 **AskQuestion**：把上述方案做成选项；自定义项单独可选或配合文字回复。

示例（仅示意；**跨类别**，且含 fit 说明）：

- A. **主从编辑式**：左 65% 主图区 `aspect-ratio` 贴近主图 + 右列上下两格辅图，`gap: 0`。**Fit**：主图 `cover`（比例匹配裁切少）；辅图按格区注明 cover/contain。
- B. **斜切双屏**：`clip-path` 对角分屏两图 + 叠字沿缝。**Fit**：仅当用户接受主体落在安全区；或其中一图 `contain` 嵌入梯形窗格 + 画意衬底。
- C. **胶片横条 + 底条舞台**：上区 75% 为横向窄条多格（横图友好）+ 下 25% 渐变舞台放叠字。**Fit**：竖图若进横条须标注强裁或改 **纵条胶片** 方案。
- D. **自定义**（请描述格数、比例、移动端、每张图是否必须全显）

### 按需追问（由你判断）

叠字与否及锚点（角/中）、**整体**动效气质偏好（更强/更柔）、氛围叠层色温与强度（**与格内画意动效、文学层字色分开**：格内 fx 与字色均须各自用户拍板）、是否引入 Google Fonts（**具体族名须用户确认**）、图片来源与路径、`prefers-reduced-motion` 策略；若用户希望 **减弱动效**，在 reduce 媒体查询外也相应收敛时长/幅度（仍须逐图确认摘要）。

### 示例问题（可改编）

- 这一页想传达的核心情绪是哪一两个词？
- **图 A / 图 B / 图 C**（按文件名）：各自主色偏暖还是偏冷？整体偏亮还是偏暗？有没有你想强调的一种颜色（如「只有这张是金色晨曦」）？
- 每张图是否 **必须整张可见**？若否，哪些可以接受 `cover` 大幅裁边？
- 若像素尺寸暂不可读：你是否接受先按「可能裁上下/左右」给方案，还是改传分辨率信息？
- 若已给出 A/B/C 布局选项，你选哪一项，或选 D 自定义？
- 每张图是否有现成文件？相对路径放在哪？（已列出文件大小与比例结论时请一并复述）
- 需要像唯美页那样每格一句哲学英文吗？放在中间还是某角？
- 动效希望「更明显」还是「略收一点」？（指整体气质；**每张图**仍须单独确认类型，见「每图画意动效」）
- **图 A / 图 B**（按文件名）：在扫光、光斑呼吸、微粒、色罩脉动、暗角呼吸等里，你更倾向哪个？要强/中/柔？要不要和轻微 `img` 浮动叠加？
- 有没有某格 **不要扫光**、**不要闪烁颗粒** 等禁忌？
- 文学叠字：**主字色**想用 `#` / `rgba` 具体值，还是从助手给的 2～3 套候选里选？**`text-shadow`** 要「强描边可读」还是「柔光晕」？
- 展示字体：要指定 **Google Fonts 族名**，还是只用 **Georgia / Times**？是否 **每格不同字色**？

### 门禁（重申）

**无回答 = 不实现。** **串联**：未完成「图片审查」（含体积、像素比例或用户确认尺寸未知与裁切/全显态度）与「逐张色系」前，不得输出排版 A/B/C。**实现稿门禁**：未完成「字体与文学层色彩（用户拍板）」中对字体族、主字色、`text-shadow` 层次、是否按格变色的 **用户确认** 前，不得将最终 `color` / `text-shadow` / `font-family` 写入 HTML/CSS。**实现稿门禁**：未完成「每图画意动效（用户拍板）」中对 **每一张入页图** 的动效类型/组合、强度及汇总表的 **用户确认** 前，不得将格内画意 `fx-*` 与对应 `@keyframes` 定稿写入 HTML/CSS。用户已在首轮说全所有必问点时，用一句话复述并请其「确认」；收到确认后再进入实现。

### 直接实现 / 仓库范例微调的务实处理

- 若用户 **书面要求**加快节奏（例如「对齐某仓库范例只改文案/动效强度」），可 **合并提问** 或对 **非安全底线项** 做 **逐项书面豁免**。
- **不得**跳过：**图片清单与体积/像素（或用户接受「尺寸未知 + 裁切/全显态度」的书面确认）、整张入画 vs `cover` 铺满格的态度**；其余项须在写入实现稿 CSS/HTML 前有 **用户确认** 或 **等价书面授权**。

## 审美原则（显式）

**强制**：凡按本 Skill 产出的基调页，**必须**按下列审美取向建造；不得以「省事」「更通用」为由改成极简、静态或系统模板风。Phase 0 仅决定 **素材、布局、字色、逐图动效类型** 等变量，**不改变**本条所定的气质底线。

- **华丽、富动态**：渐变、微粒高光、扫光、焦散、浮动、错频 `animation-delay`。
- **每图上层明显辅助动效**：每个图片格内须有 **绑定该格**、**易感知的** 画意动效层（类型与参数以 Phase 0 **用户确认** 为准）；**不能**仅靠整页一层动效或仅 `img` 浮动敷衍。
- **刻意不走极简**：避免大留白、无动效、Inter/Roboto/Arial 作主气质。
- **动效与层次是默认**：宁可分层调透明度，也不要「静态海报 + 一行标题」。

## 技术约束

- **单文件或少量 HTML**，样式以 **内联 `<style>`** 为主；**不强制** npm/打包。
- **视口与滚动**：基调页可以是 **`min-height: 100vh` 单屏拼图**（内容锁在一屏内），也可以是 **`main` 随内容增高、`body` 纵向可滚动** 的多行拼图；由 Phase 0 与用户 **「铺满格 vs 全图可见」** 决定，须在方案中写清。
- 图片 **相对路径**；默认格内 **`object-fit: cover`**，**若 Phase 0 约定全显**则对该图使用 **`object-fit: contain`**，letterbox 用 **渐变 / 噪点 / 微粒 / 与邻格同源色** 铺底，**不得**用大面积极简留白代替设计。`figure` / `.cell` 使用 `overflow: hidden`（`contain` 时衬底画在同一容器底色或伪元素上）。
- **`img` 与叠层（两种合法模式，按 fit 选用）**  
  - **铺满格**：`img` 常用 **`position: absolute; inset: 0`** + `width/height: 100%`，配合 **`object-fit: cover`**（或 **`contain` + 上条衬底**）。  
  - **全图不裁、随内容增高**：`img` 可用 **流式** **`width: 100%; height: auto`** 撑高 `figure`，格内 **`fx-*` 与叠字** 仍为 **`position: absolute; inset: 0`** 相对于该格，盖满图画区域。  
- **`z-index` 与堆叠**：`img` **垫底**（如 `z-index: 0`）；格内画意 **`fx-*` 建议 `z-index: 10+`**，确保在图之上、易被感知；**文学叠字**须 **明显高于** 画意层（如 **`z-index: 50`**）。`.cell` 可使用 **`isolation: isolate`** 稳定本格叠放上下文。
- **`contain` / 自然高度 + `img` 浮动**：若图用 `contain` 或 `height: auto`，浮动动画中的 **`scale` 宜略收敛**，避免短暂露边；与 Phase 0 确认的动效强度一致。
- 动效优先 **CSS**；装饰层 `pointer-events: none`，重要图保留 `alt`。
- **画意动效作用域**：格内 `fx-*` 须 **`position: absolute; inset: 0` 相对于该 `figure.cell`（或等价容器）**，不得把本应逐格的画意动效只写在 `body` / `main` 全屏一层代替各格（除非用户 Phase 0 明确要求仅全局氛围）。
- **画意动效定稿**：每格动效类型、层数、时长与 **用户确认的汇总表** 一致；本文件示例 `animation` / `@keyframes` **仅为模板**。
- **字体**：常见骨架为 **Georgia** + 可选 **Google Fonts** 展示衬线；**具体族名、字重、是否混排** 须在 Phase 0 经 **用户确认** 后再写入实现稿。引入外链字体时使用 `preconnect` + `display=swap`。
- **文学层色彩**：`color` 与 `text-shadow` 的 **最终数值** 以 **用户给定或确认的候选** 为准；「叠字配方」中的示例色值 **仅为演示**，不得默认当作项目定稿。

## 布局模式

### 通用

- **基底高度**：单屏拼图可用 `min-height: 100vh`；可滚动长页则 **`main` 高度随内容**，Grid 行可用 **`auto`**，配合 **`body { overflow-y: auto }`**。Grid 用 `minmax(0, 1fr)` 防止撑破（单屏行轨仍适用）。
- 典型结构：`main` 网格 + 多个 `figure.cell` 或 `section.panel`。
- 无缝拼图：`gap: 0`；需要缝时用实色/渐变间隙，**避免**用 `backdrop-filter` 做模糊接缝（除非用户明确要求）。
- **选型与审核联动**：布局类型须服务于 **图源宽高比** 与 **是否全显**；下列每条均可作为 Phase 0 的独立方案候选。

### 布局类型词典（选项池）

1. **非均分主从（编辑式）**：一侧大块主图（约 55%–70%），另一侧上下小格或竖条。**比例**：主区 `aspect-ratio` 或行高贴近主图利于 `cover` 少裁。**移动端**：可改为上下堆叠，主图在上。
2. **黄金比 / 自定义非对称分屏**：如 `1.618fr 1fr` 左右或上下分。**比例**：可消化两张横竖差异大的图。**移动端**：改为单列纵排，注意第二张变为「下屏」全宽。
3. **斜切 / 多边形（`clip-path`）**：对角或折线分屏，或底图全幅上叠梯形窗。**比例**：裁切多，须在审核中确认主体安全区或用 **contain 小窗**。**移动端**：可改为直边分屏以保可读。
4. **叠层景深**：某格负 margin 叠于另一格之上，轻阴影。**比例**：底层仍可单独做比例匹配；上层常为小卡 **contain**。**移动端**：减少重叠幅度以免遮挡触控区（若可点）。
5. **胶片横条 / 横向卷轴条**：窄横条多图并排，上下为大块画意区或叠字。**比例**：适合 **横图**；竖图需标注强裁或改用 **纵条**。**移动端**：`scroll-snap-x` 或改为竖向列表。
6. **中轴画框（门洞）**：中央固定比例画框（如 4∶5）承载主图，四周衬底。**比例**：框比 **对齐主图** 最易全显。**移动端**：框可占满宽，衬底缩为上下条。
7. **圆 / 椭圆视窗**：大圆角或正圆裁切。**比例**：装饰性强、**裁切多**，须在审核中标明「开窗」语义。**移动端**：通常保留，注意圆直径不小于关键细节可读。
8. **纵向卷轴多段**：多节 `min-height: 100vh` 串联，每节独立网格与动效（非幻灯分页、无演讲结构）。**比例**：每段可单独匹配一张或多图，适合 **比例混乱** 素材。**移动端**：天然纵滑，注意每段首屏信息密度。
9. **角标 / L 形 / T 形格**：非均匀宫格，用大角块 + 窄条消化横竖混合。**比例**：为每张图指定「主块」以贴近其比例。**移动端**：常退化为单列或 2 行堆叠。
10. **文字主导区（字即画面）**：约 40%–50% 高（或宽）为排版区，图在余下区域或背后 blend。**比例**：图区可 **contain** + 华丽衬底。**移动端**：字区与图区上下互换亦可。
11. **对称双飞**：左右对仗或镜像（镜像慎用语义与版权）。**比例**：两图宽高比宜接近。**移动端**：改为上下对仗。
12. **底部舞台条**：上 75%–85% 为画面区，底条实色/渐变承托叠字。**比例**：上图区单独设 `aspect-ratio` 或 fr 以匹配主图。**移动端**：底条可加高以免叠字挡主视觉。
13. **上通栏 + 下 2×2 四宫**：首行 **单列通栏跨两列**（常放横图），下 **两行 × 两列** 共四格。**行高**：可用 **`fr` 比例**（例如约 **38% / 31% / 31%** 对应「上约 38%、下块约 62% 且两等分」），或 **`grid-template-rows: auto auto auto`** 配合 **整页纵向滚动** 与「全图不裁」fit。**Fit**：通栏与横图比例接近时 `cover` 裁切较少；下方四格内竖图易 **上下裁**，须按「整张入画与 object-fit 策略」表与用户对齐 `cover` / `contain`+衬底。
14. **等分四宫格（2×2 全屏）**：`grid` **两行两列均分**（如 `1fr 1fr` × `1fr 1fr`），常配合 **`html, body { height: 100%; overflow: hidden }`** 锁一屏。**适用**：多张图 **并列叙事、气质可相邻**；四格 **裁切量一致**（均为 `cover` 时），便于统一节奏。**可选「联画一体」装饰**：沿 **十字缝** 铺 **窄条模糊渐变**（`linear-gradient` + `filter: blur` + 低不透明度脉动），柔化邻格色差；**中心交汇** 可叠 **光球 / 细轨道 / 星尘微粒**（`pointer-events: none`），强化四象限向心感。**叠字**：格内常 **居中纵排**（符号装饰 + 标题 + 副标 + 段落），与 Phase 0 确认层级与字号。

## 动效配方

### 叠层顺序（约定）

- **`img`**：`z-index: 0`（**流式** `height: auto` 或 **绝对铺满** 均可），保证后续层叠在画面上。
- **氛围层（格内画意）**：`position: absolute; inset: 0`，命名建议 `fx-*`；**建议 `z-index: 10+`**，**每个含图的格至少一层**，叠在 `img` 之上、文学叠字之下（避免与 `z-index: 2` 过低被感知为「在图下」）。
- **文学叠字**：最高可读层，**建议 `z-index: 50` 左右**，高于全部常规 `fx-*`。

### 画意动效类型词典（选项池）

以下为 Phase 0 **对话候选**，须 **多样选用、逐图确认**，避免全页同质化。每条：**名称 — 效果 — 常见 CSS 手段**（实现时配合 `mix-blend-mode` / 透明度）。

| 名称 | 效果（简述） | 常见手段 |
| ---- | ------------ | -------- |
| 径向光斑呼吸 | 光心在角/边缓慢胀缩、透明度脉动 | `radial-gradient` + `opacity` / `transform: scale` |
| 光斑漂移 | 高光椭圆缓慢平移或改 `background-position` | 同上 + `background-position` 或 `translate` |
| 线性扫光 | 一束亮光横/斜扫过画面 | `linear-gradient` + 伪元素 `translate` 或 `background-position` 动画 |
| 斜向掠光 | 窄条高光斜穿 | 细条伪元素 + `rotate` + `translate` |
| 色罩脉动 | 整格冷暖/单色半透明层呼吸 | 纯色或渐变层 `opacity` 动画 |
| 冷暖交替罩 | 两层相反色温低不透明度切换 | 两层 `fx-*` 交替 `opacity` 或单层渐变 `background` 动画 |
| 微粒 / 星尘 | 细小亮点慢漂 | 多 `box-shadow` 或小伪元素 + 微位移 |
| 暗角呼吸 | vignette 加深变浅 | `radial-gradient`（黑/深色系）`opacity` 或 `scale` |
| 细噪闪烁 | 胶片感颗粒轻闪 | 低 `opacity` 噪点纹理层 + 极弱 `opacity` 动画 |
| 水波慢摆 | 柔化波纹感 | `soft-light`/`overlay` + 渐变或伪元素慢摆 |
| 金纹 / 丝缕 | 细线或窄渐变条微移 | `linear-gradient` 细条 + `translate` / `background-position` |
| 边框微光 / 缝光 | 沿格边缘或内缝扫光 | `inset` `box-shadow` 或伪元素贴边动画 |
| 焦散感高光 | 多小块高光错频闪动 | 数个小 `radial-gradient` 伪元素 + 错频 `animation-delay` |
| 图片轻浮动 | 图本体微升落与略放大 | `img` 上 `translateY` + `scale`（可与格内 `fx-*` **叠加**，**不能**单独替代格内画意层） |
| 飘落轨迹（瓣/屑/字符） | 自上而下、带横移与旋转的循环 | 多子元素 `linear` + 各不相同的 `duration` / `delay` / 水平起点；可用 **少量 JS** 随机化参数以实现 **自然错频**（仍须 Phase 0 同意是否用脚本） |
| 伪元素叠层光斑 | 格容器 `::before` 承载多层径向光 | 多层 `radial-gradient` + **`opacity` 脉动** 或 **`background-position` 微移**（与「径向光斑呼吸 / 光斑漂移」可合并选型） |
| 色罩色相漂移 | 整体冷暖微摆 | 单层或伪元素上 **`filter: hue-rotate` + `brightness`** 慢变（**注意性能**；弱机或 Phase 0 要求「静」时慎用或降级为纯 `opacity`） |
| 缝间模糊连接器 | 格与格交界处柔化拼缝 | **跨缝**细条 `linear-gradient`（取邻格色）+ **`blur` + `opacity` 脉动**；**整页级**装饰，**不是**替代各格 `fx-*` |
| 中心交汇焦点 | 四宫或十字布局的向心装饰 | 绝对定位于视口中心：`radial-gradient` 光球 **`scale`/阴影脉动**、**细圆环**慢旋、中心微粒 **`dustFloat` 类漂移**；默认 `pointer-events: none` |
| 格悬停强调 | 某一格短暂「浮起」 | `:hover` 略 **`scale`**、抬 **`z-index`**、**外发光**；若需可点进剧情须 Phase 0 确认，且 **移动端** 需等价反馈（如 `:focus-visible`）或降级 |

**组合**：同一格可叠 **2 层弱效**（如微粒 + 暗角呼吸），须在 Phase 0 汇总表中写清，避免过载与用户未同意的闪烁。

### 每图独立画意层（技术约定）

- **绑定**：每格 `fx-*` 的包含块必须是 **该格容器**（`.cell` / `figure`），`inset: 0` 贴齐本格，**不得**用单层全屏元素替代多格独立画意。
- **明显可感知**：动效应在静看数秒内可被察觉（用户要求「极柔」时除外，但须 **书面确认**）。
- **可见度（易被看见）**：除「能感知」外，静态一帧上也应能看出层次——**不透明度、`mix-blend-mode` 组合** 不得弱到「像没有层」。若用户反馈 **看不见画意**，优先 **提高层不透明度、调整 blend、略加大动画幅度或延长周期**，而不是撤掉格内 `fx-*`（除非用户书面要求）。
- **错频**：各格 **不同** `animation-duration`、**负** `animation-delay`，优先 **不同 `@keyframes` 名称**，避免整页同相位复制粘贴。**同类粒子很多份**（如飘落物）时：除手写多组 `nth-child` delay 外，可用 **脚本为每份元素随机 `animation-duration` / `delay` / 起点**（仓库「等分四宫 + 飘落」类页常用），仍须 **Phase 0 确认**是否允许 JS 与粒子密度上限。
- **禁止**：仅用 `body`/`main` 一层盖全页而 **各格内无** `fx-*`；或多数格静态、仅个别格有画意层（除非用户声明）。
- **用户拍板**：见 Phase 0「每图画意动效（用户拍板）」；未确认不得实现定稿。

### `mix-blend-mode` 速查

| 模式 | 常见用途 |
| ---- | -------- |
| `screen` | 亮光、霞光、高光粒子 |
| `soft-light` / `overlay` | 统一色调、水波、金纹 |
| `multiply` | 压暗、暖色罩、晚霞厚重感 |
| `normal` | 需明确不混合的微粒（慎用透明度） |

### 节奏

- 每区 **不同** `animation-duration` 与 **负** `animation-delay`，避免整页同相位；**优先**配合 **不同 keyframes** 以拉开气质。
- **入场**：多格可同时用 **同一 keyframes**（如上移 + 淡入），但 **`animation-delay` 按格递增**（如 0.2s、0.4s…）形成 **阶梯显现**；须在 Phase 0 确认是否要「首屏编排感」。
- 图片微动：`translateY` + `scale` 略大于 1；**`contain` 或 `height: auto` 时 `scale` 宜更克制**，避免浮动露边；与格内 `fx-*` 时长 **错开** 更佳。

## 叠字配方（`.cell-verse` 类模式）

**原则**：下列为 **可读性手法** 与 **历史上常用的气质示例**，**不是**默认配色。具体 **色相、透明度、shadow 每层颜色** 必须来自 Phase 0 **用户拍板**（参见「字体与文学层色彩」）；可按主题从该节的 **类型代号表** 中选方向再落实数值。

- **英文 + `font-style: italic`**（若用户要中文叠字可改，仍须用户确认字重与字体）。
- 哲学短句，**每格一句**（除非用户要求单句跨格）。
- **多层 `text-shadow`**：通常组合 **暗色描边 + 浅色高光/晕光 + 轻扩散**，保证叠在复杂画上可读；**层数与颜色** 随用户选定方案变化，不限于金褐色系。
- **锚点**：用 flex `align-items` / `justify-content` + `clamp()` padding；可按格覆盖（左上 / 右下 / 正中）；**按格变色** 时每格单独 `color` / `text-shadow`。
- **展示字体**：`Playfair Display`、`Cormorant Garamond`、`EB Garamond`、`Cinzel` 等 **仅为常见选项**，**以用户确认的族名为准**；与 Georgia 分工（标题/正文）亦须用户同意或指定。

**可选实现范式（通用类名示例；仍须 Phase 0 字色气质确认）**

- **玻璃卡容器**（如 `.cell-verse__glass`）：`backdrop-filter`、半透明底 + 渐变或细边框；**内边距建议用 `em`**，便于与字号联动。彩虹/玻璃感仅为 **候选风格**，不是默认定稿。
- **渐变字**（如 `.cell-verse__text`）：`background-clip: text` + `-webkit-text-fill-color: transparent` + 多层 `linear-gradient`；**描边可用 `em`**（如 **`0.03em`～`0.05em`**）随字号缩放，可读性仍须用户确认。
- **字号与玻璃同尺度**：在玻璃容器上设 **`container-type: inline-size`**，正文/标题可用 **`font-size: clamp(min, N·cqi + rem, max)`**（`N` 按面板宽度与可读性调），使字随 **面板宽度** 变化；**无容器查询时** 可用 **`clamp(rem, vw, rem)`** 等 fallback，仍须 Phase 0 确认上下限。

**演示用色（禁止未确认照抄）**：如 `color: rgba(206, 163, 61, 0.78)` 仅代表「暖金系」量级，实现时必须替换为用户确认值。

## 可访问性

- 纯装饰层加 `aria-hidden="true"`。
- **`@media (prefers-reduced-motion: reduce)`**：相关 `animation: none !important`，并给静态 `opacity` 以免闪烁残留。
- Phase 0 若用户声明 **减弱动效**（不限于系统 reduce），应在逐图确认表中记 **柔/静**，实现时缩短时长、降低幅度或改为近静态层。

## 禁忌

- 模糊接缝（`backdrop-filter` 拼缝）——除非用户明确要求。
- 与画意无关的强几何装饰（如大面积 conic 射线、与内容打架的 mask 条带）——除非用户要该语言。
- 忽略 `reduced-motion`。
- **极简模板风**（大留白、无动效、系统字体当主视觉）。
- **未经用户确认**，将本 Skill **叠字配方示例**或惯例中的 **暗金 / 单一路径色值** 直接用作全页文学层定稿色。
- **未经逐图确认** 即写入格内画意动效定稿；或 **仅用** 全屏单层 / 仅 `img` 浮动 **代替** 每格独立 `fx-*`（用户书面同意的除外）。
- 从选项池 **机械复制同一套动效** 到每一格导致全页同质化——须按图系与用户选择 **多样组合**。

## 代码片段模板

本节只固定 **六种常用结构范式**；具体数值、`@keyframes`、色与字体以 Phase 0 确认为准。**文学叠字**写法与可读性手法见上文「叠字配方」，此处不重复贴代码。

### 1. 图片垫底 + 浮动（唯美式）

- `img` 铺满格、`z-index` 垫底；可选轻量 `translateY` + `scale` 浮动。
- **须配合**格内画意 `fx-*`，**不能**单靠浮动代替「每图独立画意层」；是否启用浮动与时长见 **逐图动效确认**。

### 2. 氛围层叠在图上（唯美式 bloom）

- 在 **该 `.cell` 内** `position: absolute; inset: 0` 叠渐变层，`mix-blend-mode`（如 `screen`）+ `animation`。
- 渐变形态与动效以 **该图已确认** 方案为准，勿默认定稿或照搬其他格。

### 3. 黑深残式前景 + blend

- 前景容器（可整页或格内）包一层或多层场域（如金纹、涟漪），`soft-light` / `overlay` 等 blend。
- 若存在整页前景，**每个含图格内仍须有**独立画意 `fx-*`；层序与 blend **逐图确认**。

### 4. `contain` + 画意衬底（同一 `.cell` 内）

- `.cell` 设相对定位与画意 `background`（渐变/纹理感）；`img` 使用 `object-fit: contain`、`object-position: center`。
- letterbox 区即衬底；可按需在 `img` 与叠字之间加格内 `fx-*`。

### 5. 玻璃卡 + 渐变字 + `cqi` 字号

- 叠字外包一层 **玻璃卡**（`backdrop-filter`、半透明底/边）；卡上设 **`container-type: inline-size`**，内部标题/正文用 **`clamp(..., cqi, ...)`** 与 **`em` 内边距** 联动。
- 渐变字用 **`background-clip: text`**；描边用 **`em`** 单位随字缩放。**z-index**：文学块高于本格 `fx-*`（见「技术约束」叠层区间）。
- 色与渐变条带、是否采用该范式 **仍以 Phase 0 用户确认为准**；不可用仓库范例数值未确认照抄。

### 6. 等分四宫 + 缝间连接 + 中心焦点（联画一体）

- **每格**：`img` 垫底 → **半透明渐变罩** → **格内 `fx-*`**（径向光斑、飘落层等）→ **文学层**（常居中）；格间 **`z-index` 与 `isolation`** 按上文约定。
- **整页装饰（可选）**：十字缝 **模糊渐变条**、视口中心 **光球 + 旋转细环 + 微粒**；层默认 **`pointer-events: none`**，避免挡操作。
- **错频与脚本**：飘落物可用 **CSS 多元素** 或 **脚本批量设 `duration`/`delay`**；**悬停 / 点击涟漪 / 鼠标视差** 属增强项，须 Phase 0 确认是否需要及 **`prefers-reduced-motion` 下关闭或静态化**。

## 子模式提示

- **亮色画意**（唯美、圆满）：奶油/雾蓝底、暖色 bloom、screen/soft-light 为主。
- **暗色文学**（黑深残）：近黑底、紫金渐变、金纹与噪点、`multiply`/`overlay` 慎用对比度。

以上指 **氛围层与底调** 倾向；**文学叠字的具体颜色与字体** 仍须单独走「字体与文学层色彩（用户拍板）」，**不得**因选了某一子模式就自动套暗金或固定字体。**每格画意动效** 仍须走「每图画意动效（用户拍板）」，不得因子模式自动统一动效类型。

按用户在 Phase 0 的选择选用，不要两种硬混除非用户要求对比实验。
