// 项目概览

重新定义叙事游戏的内容生产

这不是一个简单的聊天脚本，而是一套完整的、可落地的工程化内容管线——将「写设定 → 拆章节 → 生成剧情 → 出立绘/场景」从手工拼 prompt，收束成由 AI 驱动的全自动化内容工作流。

叙事纪元的核心是一套 AI 叙事闭环：用户输入主题与偏好，引擎自动生成结构化世界观、章节矛盾、分支剧情与选项，配合 LLM 驱动的图像生成（主角立绘、场景插图），最终根据用户选择动态调整结局基调——整个过程无需人工干预，完全由 AI 驱动。

项目支持 命令行（CLI） 与 Web 前端 两种运行模式，共用同一套 src/ 核心逻辑，存档互通。Web 端还实现了两层预生成技术，显著降低玩家等待感，让沉浸体验更加流畅。

无论你是想开发一款 互动叙事游戏、AVG、跑团式文字冒险，还是构建 AI 原生游戏原型，这套引擎都能提供坚实的内容基础与工程支撑。

// 核心能力

功能特性全景

从叙事到视觉，从单轮到批量，从文本到多模态——每一项能力都经过精心设计，服务于同一个目标：让 AI 叙事游戏的内容生产变得可工程化、可重复、可扩展。

AI 驱动

LLM 叙事引擎

基于大语言模型的全自动世界观与剧情生成。支持 Claude Opus 4.6 等顶级模型，通过结构化 prompt 与解析引擎，保证生成内容的可维护性与一致性。

核心

流动世界线

独创的 flow_worldline 状态系统，实时追踪当前章节、矛盾解决状态与信息差记录，让剧情随玩家选择动态演变，而非线性预设。

AI 驱动

多模态图像引擎

支持云雾（yunwu）、Replicate、OpenAI DALL·E、Stable Diffusion、ComfyUI 等六大图像供应商，主角立绘与场景图均由 LLM 优化提示词后生成。

系统

多层预生成管线

Web 端独有的两层预生成技术：在玩家阅读当前剧情时，后台已预先生成下两层选项，选择后即时命中缓存，极大降低等待感。

核心

配角动态建档

从剧情中自动识别、提取配角信息，首次出场时触发建档，支持别名演化（如"黑衣人"→"艾玛"），维护完整的配角身份图谱。

AI 驱动

深层背景系统

角色拥有隐藏设定（深层背景），选项可关联并解锁这些秘密。解锁后剧情自动围绕已揭示内容展开，避免一次性信息轰炸。

核心

结局预测与基调修改

开局即生成隐藏的结局预测，在深层背景节点触发时动态修改 HE/BE/NE 等结局基调，让玩家选择真正影响最终走向。

AI 驱动

Wikipedia 增强

对现实题材与 IP 内容，自动调用 Wikipedia 中英文 API 检索资料，将检索结果注入 LLM 与图像提示词，增强真实性与代入感。

系统

信息差驱动剧情深化

当未发现的信息差达到阈值（如 5 条），自动触发隐藏剧情深化（揭秘/反转），将"隐藏信息→爽点释放"做成可配置的机制。

// 叙事系统

七种故事基调 · 精细化叙事规则

每种基调不仅是一句描述，而是叙事逻辑、文风、角色规则、关系处理、冲突解决、主题取向、细节要求、禁忌等多维规则，直接参与世界观与剧情 prompt 的构建。

深度圆满结局 HE

苦尽甘来型正向结局。主角历经真实挫折与痛苦完成蜕变，所有矛盾逻辑自洽地解决，导向释然与希望。

  • 先抑后扬，磨难真实有逻辑
  • 主角靠自身成长与合理助力达成
  • 所有伏笔人物动机全部回收
  • 禁止核心主角死亡、机械降神

高级悲剧结局 BE

宿命式悲剧。故事前期构建温暖与希望，最终因宿命枷锁或现实碾压，所有珍视的事物尽数崩塌。

  • 先扬后抑，阶段性胜利铺垫
  • 悲剧伏笔深埋，非突然崩盘
  • 结局不可逆，无任何挽回可能
  • 禁止伪BE、留余地、机械降神

现实向普通结局 NE

缺憾圆满式。主角达成部分核心目标，收获踏实的小圆满，同时留下合理且不可逆的遗憾。

  • 有起有落，得失均衡
  • 现实伏笔深埋，非刻意失衡
  • 落脚于释然与平凡
  • 禁止无脑圆满、无意义悲剧

黑暗宿命式结局

三体式黑暗。规则至上、宿命难违。主角奋力挣扎却始终无法挣脱规则束缚，终局开放式留白。

  • 规则立基，黑暗源于规则本身
  • 挣扎有迹，希望易碎
  • 沉思导向，拒绝宣泄
  • 禁止为黑而黑、暴力渲染

自然幽默风格

幽默统筹全局。人物自带幽默特质，笑点藏于日常与互动，轻松松弛，适配所有主题。

  • 幽默融于情节，不脱节
  • 笑点张弛有度，不密集
  • 人物幽默有边界，贴合人设
  • 禁止强行尬笑、低俗恶搞

思想超脱式抽象风格

哲学寄托型。思想跳跃与超脱，将存在与虚无、自由与束缚等哲学命题自然寄托于文字与情节之中。

  • 思想为核，不拘于线性逻辑
  • 碎片化情节与意识流思考
  • 多元视角切换
  • 拒绝行文晦涩、结构松散

// 角色系统

配角动态建档 · 深层背景解锁

角色不是一次性抛出的设定，而是一个随剧情推进逐步揭示、层层深化的动态体系。

主角设定

属性选择
用户在开局选择主角属性（性别、年龄、性格等），系统基于此生成主角规范信息（protagonist_canonical），供 LLM 剧情生成与图像提示词优化共同使用，保证叙事与视觉的一致性。
三视图立绘
依赖世界观异步生成主角正面/侧面/背面三视图，存入 initial/main_character/<game_id>/ 目录，由 LLM 优化主角形象提示词后调用图像 API 生成。
深层背景
主角拥有隐藏设定（深层背景），选择关联选项后解锁，后续剧情自动围绕已揭示深层背景展开，形成完整的角色弧光。

配角系统

动态识别与建档
从剧情中自动识别出场/提及配角（extract_supporting_characters_in_scene），首次展示后触发建档（archive_supporting_role_first_appearance），避免凭空生成。
身份揭示
从剧情中提取身份揭示（如"黑衣人就是艾玛"），自动更新配角别名与档案（update_supporting_role_aliases_from_plot），支持别名演化链。
深层背景关联
配角同样拥有深层背景，选项可关联并解锁。选择后记录信息差并触发结局基调修改，使配角成为影响故事走向的关键变量。

// 运行模式

双端一致 · 体验无损

CLI 与 Web 共用同一套 src/ 与 main2 逻辑，存档互通。Web 端通过预生成技术进一步提升体验。

命令行模式 CLI

在终端里直接进行文字冒险，适合快速体验与调试

  • 执行 python main.py 或 python main2.py 即可启动
  • Windows 下已处理 UTF-8 编码，中文正常显示
  • 共用 src/ 与 main2.py 的完整逻辑
  • 存档写入 saves/ 目录，与 Web 端互通
  • 剧情与选项通过 safe_input() 交互
  • 支持新游戏、加载、存档管理
python main.py # 启动命令行版

网页模式 Web

浏览器内游玩，配合图像与预生成提供沉浸体验

  • 执行 python game_server.py，浏览器打开 http://127.0.0.1:5001
  • Flask 提供 RESTful API，前端单页应用 game-frontend/
  • 两层预生成：用户阅读时后台已生成下两层，选择即命中缓存
  • 剧情图可选生成，失败时有纯文本兜底
  • 主角属性 + 难度 + 基调 + 主题 + 图片风格完整引导流程
  • Web 存档与 CLI 共用同一目录，跨端读取
python game_server.py # 启动 Web 版

// 预生成管线

两层预生成 · 即选即得

Web 端的核心体验优化技术：在用户阅读当前剧情时，后台已经预先生成好了接下来两层的内容，选择后即时命中，无需等待。

T+0 · 用户阅读中
当前层剧情展示

前端展示当前场景描述 + 2个选项（及可选场景图）。同时，前端请求 /pregenerate-next-layers，通知后端开始预生成。

用户感知
T+1 · 后台异步执行
预生成两层选项与剧情

后端为当前 2 个选项分别调用 generate_all_options，生成第一层 + 第二层的剧情与选项。选项可带场景图，并行生成后写入 pregeneration_cache。

异步处理 写入缓存
T+2 · 用户做出选择
命中预生成缓存

用户点击第一层选项，前端调用 /get-pregenerated-layer2，后端从缓存直接返回第二层数据（无需 LLM 调用），显著降低等待感。同时清理旧缓存与已用选项。

即时响应 缓存命中
T+3 · 循环迭代
新场景 + 新预生成

展示第二层剧情后，前端再次请求 /pregenerate-next-layers，为新的选项层预生成后续两层内容。循环往复，始终保持"已生成两层"的状态。

持续异步 循环缓存

// 创新亮点

七大创新点

这些创新不是噱头，而是经过工程验证的、能够真正提升 AI 叙事游戏质量与体验的设计。

01

流动世界线 + 章节矛盾 + 结局预测联动

将「当前章节、矛盾是否解决、信息差记录」显式纳入状态（flow_worldline），每一步剧情由 LLM 输出 flow_update；开局预先生成隐藏结局预测，深层背景节点触发时动态修改 HE/BE/NE 基调，使结局既有伏笔又具备动态调整空间。

02

信息差驱动的隐藏剧情深化

用「未发现的信息差条数」达到阈值（如 5 条）触发一次性的隐藏剧情深化（揭秘、反转），约束「自然嵌入常规剧情、不直接抛设定」，把「隐藏信息 → 爽点释放」做成可重复、可配置的机制。

03

选项与深层背景的显式绑定

在 prompt 中要求「部分选项必须关联角色深层背景」并在输出中带【深层背景关联】标记；选择后解锁角色深层背景，强制后续剧情围绕已解锁内容展开，形成可解析、可执行的闭环。

04

配角「展示后建档」+ 身份揭示

配角档案在展示后触发建档，避免未出场角色凭空生成；同时用 LLM 从剧情中抽取身份揭示（如「黑衣人就是艾玛」）更新配角别名与档案，使配角体系既动态又与剧情强绑定。

05

预生成两层 + 预生成图片的 Web 体验

在用户看到当前层时后台预生成「下一层 + 再下一层」的选项与剧情（并可带图），选择后直接命中缓存返回，把「等生成」转化为「选完即出」，且与缓存清理、线程安全一起设计。

06

多基调的细粒度叙事规则

每种基调（HE/BE/NE、黑暗宿命、幽默、唯美、逻辑推理、克苏鲁式等）不仅是一句描述，而是叙事逻辑、文风、角色规则、关系处理、冲突解决、主题取向、细节要求、禁忌等多维规则，直接参与 prompt 构建。

07

现实题材的 Wikipedia 注入

将 Wikipedia 主题与人物检索结果作为证据块注入 LLM 与图像提示词，在保持通用剧情生成的前提下，对现实/IP 题材做定向增强，兼顾通用框架与领域深度。
